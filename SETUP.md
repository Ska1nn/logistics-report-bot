# Инструкция по настройке Telegram-бота с Google Sheets

## 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

## 2. Google Sheets API - Настройка

### 2.1 Создание проекта в Google Cloud Console

1. Перейдите на [Google Cloud Console](https://console.cloud.google.com/)
2. Создайте новый проект
3. Включите API: **Google Sheets API** и **Google Drive API**
4. Создайте сервисный аккаунт:
   - В левом меню: **Сервисные аккаунты** (Service Accounts)
   - Нажмите **Создать сервисный аккаунт**
   - Заполните имя и описание
   - На шаге 2 нажмите **Создать ключ** → **JSON**
   - Загруженный файл переименуйте в `credentials.json` и поместите в корень проекта

### 2.2 Создание Google Sheets

#### Таблица 1: Справочники (Reference)
- Название: `Reference`
- Листы (с примерами данных):
  - `Материалы` - список материалов (щебень 20-40, песок, гравий и т.д.)
  - `Карьеры` - список карьеров/поставщиков
  - `Объекты` - список объектов доставки

#### Таблица 2: Результаты (Results)
- Название: `Результаты`
- Столбцы:
  1. Имя пользователя
  2. ID пользователя
  3. Дата
  4. Госномер
  5. Тип операции
  6. Поставщик
  7. Материал
  8. Объект
  9. Единица измерения
  10. Количество
  11. Время добавления

### 2.3 Предоставление доступа сервисному аккаунту

1. Откройте обе таблицы Google Sheets
2. Нажмите **Поделиться** (Share)
3. Скопируйте email сервисного аккаунта из файла `credentials.json` (поле `client_email`)
4. Вставьте email и дайте права редактора (Editor)

## 7. Структура бота

### Основные команды:
- `/start` - запуск бота
- **➕ Добавить** - начало ввода новой записи
- **📋 История** - просмотр истории (в разработке)

### Процесс добавления данных:
1. **Дата** - ДД.ММ.ГГГГ или кнопка "Сегодня"
2. **Госномер** - номер автомобиля
3. **Тип операции** - Закуп / Продажа / Перевозка
4. **Поставщик** - название карьера или поставщика
5. **Материал** - название материала (щебень, песок и т.д.)
6. **Объект** - объект доставки
7. **Единица измерения** - тонна / м³ / рейс / час
8. **Количество** - число
9. **Подтверждение** - проверка данных перед сохранением

## 8. Обработка ошибок

Если возникают ошибки:
- Проверьте, что `credentials.json` находится в корне проекта
- Убедитесь, что сервисному аккаунту дан доступ к обеим таблицам
- Проверьте токен бота в `.env`
- Посмотрите логи в консоли для деталей ошибки
